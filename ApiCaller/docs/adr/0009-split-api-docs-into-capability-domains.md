# Split API Docs Into Capability Domains

API documentation should be organized by capability domains and second-level pages instead of broad role pages. Platform configuration APIs are split into asset library, label, device, and data management pages; collection work APIs are split into project, strict task, scene task, and job pages so users can find the exact operation type and required inputs without scanning a coarse API list. The top-level API index should remain a progressive-disclosure navigation page and should not duplicate endpoint tables or field details from child pages.
