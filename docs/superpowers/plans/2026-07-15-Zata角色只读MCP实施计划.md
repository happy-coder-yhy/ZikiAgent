# Zata 角色只读 MCP 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 新增管理员 13 个、采集员 3 个精确只读工具的 stdio MCP，并在模型接触数据前完成身份注入、结构化 ToolOutcome、递归字段白名单、审计和 Token 隔离。

**Architecture:** 现有注册函数通过一个公开装饰器兼容的捕获器提供 legacy 只读 handler，不创建完整 MCP，也不访问 FastMCP 私有注册表。外层 ReadToolGateway 在调用前再做角色授权，调用后把不可信 JSON 映射为白名单 `ToolOutcome`；采集员身份敏感工具使用显式安全包装。

**Tech Stack:** MCP Python SDK/FastMCP、Pydantic、现有 ZataAPICaller、标准库 JSON/NDJSON、unittest。

```python
class ToolOutcomeStatus(StrEnum):
    SUCCESS = "success"
    NEEDS_INPUT = "needs_input"
    MULTIPLE_CANDIDATES = "multiple_candidates"
    NOT_FOUND = "not_found"
    PERMISSION_DENIED = "permission_denied"
    PLATFORM_ERROR = "platform_error"
    TIMEOUT = "timeout"


class ToolOutcome(BaseModel):
    model_config = ConfigDict(extra="forbid", frozen=True)

    schema_version: Literal["1.0"] = "1.0"
    status: ToolOutcomeStatus
    safe_data: JsonValue | None = None
    error_code: str | None = None
    safe_message: str
    candidates: tuple[ToolCandidate, ...] = ()
    trace_id: UUID
    retryable: bool = False
```

`model_validator` 强制 multiple candidates 有候选；needs input 有问题/必填字段；permission/platform/timeout 有稳定错误码。枚举中不存在 `awaiting_confirmation`。

- [ ] **Step 4: 实现资源上限和秘密清理**

`GatewayLimits` 默认：原始响应 1 MiB、嵌套 8 层、数组 200 项、候选 20 项、单字符串 4,000 字符。超限结果拒绝或安全截断，不复制被截内容。

`SecretRedactor` 对键名大小写/分隔符归一化，拒绝 `token/accessToken/access_token/authorization/cookie/password/secret/headers/raw/raw_text/_warnings/img`。对字符串中的 Bearer、Cookie 和当前运行 Token 指纹做替换；异常对象只记录内部分类，不把 `str(e)` 放入 ToolOutcome。

```python
def parse_legacy_json(raw: str, limits: GatewayLimits) -> Mapping[str, JsonValue]:
    encoded = raw.encode("utf-8")
    if len(encoded) > limits.max_response_bytes:
        raise UnsafeToolResponse("response too large")
    value = json.loads(raw)
    if not isinstance(value, dict):
        raise UnsafeToolResponse("top level must be an object")
    return value
```

公开工厂固定为 `success()`、`needs_input()`、`multiple_candidates()`、`not_found()`、`permission_denied()`、`platform_error()`、`timeout()`；所有用户可见文本由稳定模板构造。

- [ ] **Step 3: 实现最小捕获器**

```python
class CapturingToolRegistry:
    def __init__(self, allowed: frozenset[str]) -> None:
        self._allowed = allowed
        self._handlers: dict[str, Callable[..., str]] = {}

    def tool(self):
        def decorator(function: Callable[..., str]) -> Callable[..., str]:
            if function.__name__ in self._allowed:
                if function.__name__ in self._handlers:
                    raise ToolRegistrationError("duplicate tool")
                self._handlers[function.__name__] = function
            return function
        return decorator
```

调用现有 admin/collector `register_tools(registry, caller)` 后，对预期名称做严格相等检查。写 handler 虽会在注册函数局部定义，但从不保存、注册或暴露。

- [ ] **Step 4: 处理 upstream 观察而不改 legacy 返回契约**

给捕获 handler 传入 `ObservingCaller` 代理。代理转发 `ZataAPICaller` 公共方法，只记录本次调用的 HTTP 状态、业务 code、超时或异常分类，不记录响应体/请求头/Token。这样 `job_summary` 的全零结果能够区分“真实空数据”和“所有平台调用失败”。

- [ ] **Step 3: 实现 Protocol 和首期进程内 resolver**

`CredentialResolver` 定义 `issue/resolve/revoke`；`InMemoryCredentialResolver` 由 composition root 显式创建，使用锁保护映射，引用值由 `secrets.token_urlsafe(32)` 生成。Token 使用不可打印的 `SecretStr` 容器，所有异常只含安全错误码。
