# Zata 运行编排、恢复与取消实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不依赖 Hermes 内部 API 的前提下，实现单 Session 单活动 Turn、结构化补充信息恢复、单调事件、独立 Worker、超时与 Windows 完整进程树取消。

**Architecture:** `ExecutionCoordinator` 负责数据库状态机和凭据生命周期，`ProcessAgentRuntime` 负责每个 run 的独立进程，Worker 通过稳定 `AgentBackend` 协议调用最终适配器。MCP Gateway 写入 run-scoped outbox，父进程实时摄取并持久化产品事件/工具审计；取消使用数据库状态、取消标记文件和进程树终止三层保证。

**Tech Stack:** SQLAlchemy、multiprocessing spawn、ThreadPoolExecutor、psutil、cryptography/Fernet、标准库 queue/locking、unittest。
