# Zata 智能体执行层总路线图

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不删除、不重命名、不削弱现有 28 个 MCP 工具和 legacy 启动方式的前提下，新增一条角色隔离、只读、可审计、可取消、可补充信息恢复的 Web Agent 执行路径。

**Architecture:** 新路径由 FastAPI Web 边界、Execution Coordinator、稳定的 `AgentRuntime` 协议、Hermes Python 适配器、每次运行独立 Worker、角色只读 stdio MCP 和每实例独立 `ZataAPICaller` 组成。产品数据库持久化会话、历史、事件、审计和恢复状态；Hermes 的 Memory、trajectory、上下文文件和自修改能力在首期关闭。

**Tech Stack:** Python 3.11–3.13、Hermes Agent 0.18.2、MCP Python SDK 1.27.2、FastAPI 0.136.3、Pydantic 2.13.4、SQLAlchemy 2.0.50、SQLite（开发/测试）、PostgreSQL（生产）、cryptography、psutil、unittest、uv。

## 1. 当前代码现状

仓库基线：分支 `dev-v0.0.1`，审计时提交 `d2710af`。

当前调用链只有：

```text
run_mcp_server.py
  -> mcp_server.server.create_app()
  -> 同一个 FastMCP 应用
  -> 同一个 ZataAPICaller
  -> Zata 平台
```

当前目录中不存在 Web API、Agent Runtime、数据库、会话锁、审计存储、恢复机制或进程管理器。依赖也没有通过 `pyproject.toml`/锁文件声明。

### 1.1 必须保留的 legacy 能力

- `run_mcp_server.py` 及其 stdio/SSE 启动参数。
- `python -m mcp_server` 启动方式。
- `mcp_server.server.create_app()` 的全量应用工厂语义。
- `ZataAPICaller` 及所有现有 API 封装。
- 当前 Skills、测试和全部 28 个 MCP 工具。
- 12 个写工具继续存在于 legacy 全量 MCP，但永远不注册到新 Web Agent。

### 1.2 新路径的精确工具边界

管理员只读工具必须精确为 13 个：

```text
get_platform_config
get_scene
get_task_purpose
search_user
get_projects
get_scene_task
task_summary
task_detail
job_summary
job_detail
task_job_details
device_summary
device_detail
```

采集员只读工具必须精确为现有 3 个工具，不重命名：

```text
query_task_job
query_my_device
query_device_binding
```

采集员可见签名不能出现 `collector_id`。`query_task_job` 和 `query_my_device` 的用户 ID 由服务端注入；`query_device_binding` 只能查询当前用户已绑定设备，不能成为任意设备探测入口。

### 1.3 已确认风险

1. `mcp_server/server.py` 末尾的 `mcp = create_app()` 会在导入时读取凭据、创建 Caller，并可能登录联网或 `sys.exit()`。
2. `mcp_server/__init__.py` 和 `mcp_server/__main__.py` 又直接导入该单例，放大了导入副作用。
3. legacy 工具直接返回 JSON 字符串，常见 `str(e)`、原始平台响应和整对象透传；新路径不能把这些数据直接交给模型或 Web。
4. `query_task_job(collector_id=...)` 和 `query_my_device(collector_id=...)` 允许调用方覆盖身份。
5. `query_device_binding` 当前可查询非本人设备，并会调用管理员用户列表做名称补全。
6. 旧 SSE 默认监听 `0.0.0.0`，没有发现可供 Web Agent 依赖的角色隔离边界。
7. 仓库中存在明文测试凭据；当前树必须脱敏，真实凭据必须由平台侧轮换，Git 历史不能在未获授权时改写。
8. Hermes 是当前解释器中的偶然全局安装，仓库自身没有依赖声明或适配代码；实现不能依赖开发机状态。

### 1.4 实施前测试基线

从仓库根目录、使用组合 `PYTHONPATH` 运行：

```powershell
$env:PYTHONPATH=((Get-Location).Path + ';' + (Join-Path (Get-Location).Path 'ApiCaller'))
python -m unittest discover -s ApiCaller\tests -p 'test_*.py' -v
```

审计结果：34 项中 32 通过、2 项失败。失败分别来自测试输入与断言不一致，以及场景任务创建 URL 的旧断言与当前 OpenAPI/实现不一致。

MCP 测试使用假凭据隔离导入副作用后运行 51 项，其中两个场景任务测试遗漏新增的 `collectors=None`、`auditors=None` 参数；另一个“默认凭据”测试依赖本机 `.env` 并包含明文测试凭据。第一阶段要先把这些旧测试改为确定、无密钥的基线，不能把它们记作新执行层回归。

## 2. 锁定后的目标架构

```text
Browser
  -> FastAPI Web API
     -> trusted request identity adapter
     -> ExecutionCoordinator
        -> SQLAlchemy repositories / session lease / events / audit
        -> CredentialResolver (opaque, short-lived reference)
        -> AgentRuntime protocol
           -> ProcessAgentRuntime
              -> one spawned Worker per run_id
                 -> HermesAdapter
                    -> one AIAgent per run_id
                    -> one role-scoped stdio MCP child
                       -> ToolOutcomeGateway
                       -> one ZataAPICaller
                       -> Zata platform
```

关键生命周期：

```text
session_id = 一段会话
turn_id    = 一次用户逻辑任务；等待补充与恢复期间不变
run_id     = 一次实际执行尝试；每次恢复都创建新值
```

状态边界：

```text
AgentRunResult: completed | awaiting_input | failed | cancelled
ToolOutcome: success | needs_input | multiple_candidates | not_found
             | permission_denied | platform_error | timeout
```

首期没有 `awaiting_confirmation`，因为新 Agent 路径没有写工具。

## 3. 安全决策

- Web 请求体不接受 `role`、`user_id`、平台 Token 或任意 `allowed_tools`；它们只来自可信认证适配器和服务端常量。
- `AgentRunContext` 仅含 `credential_ref`，不含原始 Token。
- Token 只在 Coordinator 解析后进入本次 Worker/stdio MCP 的受控环境，执行结束立即撤销引用。
- 每个 Worker 使用独立临时 `HERMES_HOME`；Hermes 配置文件只写 `${ZATA_ACCESS_TOKEN}` 引用，不写真实值。
- Hermes 使用 `quiet_mode=True`、`skip_context_files=True`、`skip_memory=True`、`save_trajectories=False`；只启用该次只读 MCP 工具集。
- MCP 工具发现后、创建 AIAgent 前，必须校验角色工具名称与 13/3 常量完全相等；数量或名称不符即 fail closed。
- Gateway 对 legacy JSON 做格式校验、大小/深度/数组/字符串限制和字段白名单重建；绝不把原始响应或 `str(e)` 返回给模型。
- Tool 审计在 Gateway 边界实时产生，不能靠 Hermes 最终 messages 反推。
- Web 只发布产品事件，不发布系统提示词、隐藏思维链、Hermes 内部推理或原始平台响应。
- 同一 `session_id` 始终只有一个未结束 `turn_id`；运行租约由数据库条件更新获得，不能依赖进程内锁。
- 取消先写 `cancelling`，再设置运行级取消标记并终止 Worker 及完整子进程树；重复取消必须幂等。
- 恢复 Token 只向前端返回一次，数据库只存 SHA-256 摘要；真实恢复上下文使用服务端密钥加密并绑定用户、会话、Turn、前一次 Run、候选集合和过期时间。

## 7. 不能伪装为代码已完成的外部事项

- 已暴露凭据的真实轮换必须由有平台权限的人完成；代码只能删除当前树中的明文并阻止再次提交。
- Git 历史改写会影响所有协作者，未经明确授权不执行；计划只做历史扫描并生成不含秘密值的受影响路径/提交清单。
- 当前远端是内部 Git 服务，仓库没有 CI 配置。计划会提供可直接在 CI 调用且失败即返回非零码的 `scripts/secret_scan.py`；具体 CI 平台的流水线文件必须在确认平台与现有流水线约束后接入，不能凭空创建。
- 生产数据库锁依赖 PostgreSQL；SQLite 只用于单进程开发与测试，不作为多实例生产租约实现。
